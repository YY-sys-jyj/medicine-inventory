# 医药库存动销与回款管理系统 - 运维说明

## 自动提醒

- 推送时间：GitHub Actions 每 10 分钟检查一次；真正发送时间由每个用户在“通知中心 -> 自动提醒时间”里自行设置，默认北京时间 08:00、17:00。
- 触发位置：`.github/workflows/daily-reminders.yml`
- 推送接口：`supabase/functions/wxpusher/index.ts`
- 认证方式：GitHub Secret `CRON_SECRET` 必须与 Supabase Edge Function Secret `CRON_SECRET` 一致。
- 推送对象：系统按 `user_id` 分开生成提醒，只推给该用户绑定的 PushPlus/WxPusher。
- 推送内容：手机端按类别合并，同一时段最多发送“药品效期汇总”和“医院回款汇总”两条业务提醒；网页通知中心保留单条明细。

## 客户绑定 WxPusher

客户不需要进入你的 Supabase、GitHub 或 WxPusher 管理后台。客户只需要：

1. 打开或下载 WxPusher。
2. 找到自己的 `UID_xxxxx`。
3. 登录本系统，进入“通知中心 -> 微信提醒绑定”。
4. 把自己的 UID 填入“客户自己的 WxPusher UID”，保存绑定。
5. 点“发送测试”，能收到测试消息就说明绑定成功。

之后系统会按客户自己的账号数据和提醒时间自动推送。不同客户的数据、UID 和提醒时间按 `user_id` 分开，不会互相串消息。

## 自定义提醒时间 SQL

如果通知中心保存提醒时间时报“缺字段”，在 Supabase SQL Editor 运行：

```sql
alter table public.wechat_bindings
add column if not exists reminder_morning_time text not null default '08:00';

alter table public.wechat_bindings
add column if not exists reminder_evening_time text not null default '17:00';

alter table public.wechat_bindings
add column if not exists reminder_morning_enabled boolean not null default true;

alter table public.wechat_bindings
add column if not exists reminder_evening_enabled boolean not null default true;

do $$
begin
  if not exists (
    select 1 from pg_constraint
    where conname = 'wechat_bindings_reminder_morning_time_format'
  ) then
    alter table public.wechat_bindings
    add constraint wechat_bindings_reminder_morning_time_format
    check (reminder_morning_time ~ '^([01][0-9]|2[0-3]):[0-5][0-9]$');
  end if;

  if not exists (
    select 1 from pg_constraint
    where conname = 'wechat_bindings_reminder_evening_time_format'
  ) then
    alter table public.wechat_bindings
    add constraint wechat_bindings_reminder_evening_time_format
    check (reminder_evening_time ~ '^([01][0-9]|2[0-3]):[0-5][0-9]$');
  end if;
end $$;
```

## 数据保留

定时推送执行时会顺手清理过期运营数据：

- 通知记录：保留 90 天。
- PushPlus 临时扫码绑定会话：保留 30 天。
- 出入库日志、回款日志、删除日志：保留 365 天。

业务台账数据不会被自动清理。
